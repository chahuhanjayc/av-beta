## 1. Project Overview

Akshaya Vistara is a production-ready multi-tenant accounting system built with Django.
It enforces strict double-entry bookkeeping, automates bill entry with OCR,
and provides financial reports — all behind a clean Bootstrap 5 SaaS UI.

**Phases built:**
| Phase | Features |
|-------|----------|
| 1 | Multi-tenancy, Custom Users, Ledgers, Vouchers with double-entry enforcement |
| 2 | OCR bill scanning (Tesseract), auto-create Purchase Vouchers, duplicate detection |
| 3 | P&L Report, Balance Sheet, Receivables Aging, Cash Flow Chart, Docker, PWA |

## 4. Development Setup (Windows 11)

### Prerequisites
- Python 3.11+ — https://www.python.org/downloads/
  - ✅ Check "Add Python to PATH" during install
- Git — https://git-scm.com/download/win (optional)

### One-Click Setup

Open **PowerShell** (not Command Prompt), navigate to the project folder, then:

```powershell
# Step 1 — Allow script execution (one-time, per machine)
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

# Step 2 — Run the setup script
cd "C:\path\to\akshaya_vistara"
.\setup.ps1
```

`setup.ps1` automatically:
1. Creates a Python virtual environment (`.venv`)
2. Installs all packages from `requirements.txt`
3. Copies `.env.example` → `.env` with a randomly generated `SECRET_KEY`
4. Runs all database migrations
5. Collects static files
6. Prompts you to create a superuser (admin account)
7. Offers to start the development server

### Every time after setup

```powershell
cd "C:\path\to\akshaya_vistara"
.\.venv\Scripts\Activate.ps1
python manage.py runserver
```

Open **http://127.0.0.1:8000** in your browser.

## 5. Environment Variables Reference

Copy `.env.example` to `.env` and set these values:

| Variable | Required | Description |
|----------|----------|-------------|
| `SECRET_KEY` | ✅ | 50+ character random string. Auto-generated by setup.ps1 |
| `DEBUG` | ✅ | `True` for development, `False` for production |
| `ALLOWED_HOSTS` | ✅ | Comma-separated. e.g. `127.0.0.1,localhost` |
| `DATABASE_URL` | ✅ | SQLite: `sqlite:///db.sqlite3`  PostgreSQL: `postgres://user:pass@host:5432/dbname` |
| `TESSERACT_PATH` | OCR | Full path to `tesseract.exe`. Default: `C:\Program Files\Tesseract-OCR\tesseract.exe` |
| `POPPLER_PATH` | PDF OCR | Path to Poppler `bin` folder. Default: `C:\poppler\Library\bin` |

For Docker, copy `.env.docker.example` → `.env.docker` instead.

## 6. Tesseract OCR Installation

Tesseract is required for the OCR / Bill Scanning feature.

### Step 1 — Download installer

Go to: **https://github.com/UB-Mannheim/tesseract/wiki**

Download the **Windows 64-bit installer** (latest version, e.g. `tesseract-ocr-w64-setup-5.5.0.20241111.exe`)

### Step 2 — Install

1. Run the downloaded `.exe`
2. Keep the default install path: `C:\Program Files\Tesseract-OCR\`
3. ✅ Check **"Add to PATH (for all users)"** if given the option

### Step 3 — Verify

Open a **new** PowerShell window and run:

```powershell
tesseract --version
```

Expected output: `tesseract 5.x.x`

### Step 4 — Update .env (if different path)

If you installed to a different location, add to your `.env`:

```
TESSERACT_PATH=C:\Your\Custom\Path\tesseract.exe
```

## 7. Poppler Installation (PDF Support)

Poppler is required to scan PDF invoices (convert PDF pages to images before OCR).

### Step 1 — Download

Go to: **https://github.com/oschwartz10612/poppler-windows/releases/latest**

Download the file named: `Release-xx.xx.x-0.zip`

### Step 2 — Extract

Extract the ZIP to `C:\poppler\`

You should now have: `C:\poppler\Library\bin\pdfinfo.exe`

### Step 3 — Add to PATH permanently

Open PowerShell **as Administrator** and run:

```powershell
$oldPath = [System.Environment]::GetEnvironmentVariable("Path", "Machine")
$newPath = $oldPath + ";C:\poppler\Library\bin"
[System.Environment]::SetEnvironmentVariable("Path", $newPath, "Machine")
```

Restart PowerShell, then verify:

```powershell
pdfinfo --version
```

### Step 4 — Update .env (if different path)

```
POPPLER_PATH=C:\poppler\Library\bin
```

## 9. Production — Docker Setup

### Prerequisites

- **Docker Desktop** — https://www.docker.com/products/docker-desktop/
  - Enable WSL 2 backend during setup (recommended for Windows)
- Docker Desktop must be running (green icon in system tray)

### Step 1 — Create production .env.docker

```powershell
Copy-Item ".env.docker.example" ".env.docker"
notepad .env.docker
```

Fill in **all** values, especially:
- `SECRET_KEY` — generate one: `python -c "import secrets; print(secrets.token_urlsafe(50))"`
- `POSTGRES_PASSWORD` — use a strong random password
- `DATABASE_URL` — must match POSTGRES_* values, with host = `db`
- `ALLOWED_HOSTS` — your server's domain or IP

### Step 2 — Build and start containers

```powershell
# Build images and start all services in background
docker-compose up --build -d

# Watch the logs
docker-compose logs -f web
```

### Step 3 — Create superuser

```powershell
docker-compose exec web python manage.py createsuperuser
```

### Step 4 — Access the application

Open **http://localhost** (port 80) in your browser.

### Managing containers

```powershell
# Stop everything
docker-compose down

# Stop + delete all data (WARNING: destroys DB)
docker-compose down -v

# Restart a single service
docker-compose restart web

# View logs for a specific service
docker-compose logs -f nginx

# Check service status
docker-compose ps
```

### Updating the application

```powershell
# Pull latest code changes (if using git)
git pull

# Rebuild and restart
docker-compose up --build -d

# Run any new migrations
docker-compose exec web python manage.py migrate
```

## 10. Database Backups

```powershell
# Create a backup (saves to .\backups\backup_YYYY-MM-DD_HH-MM-SS.sql)
.\backup.ps1

# Keep 14 most recent backups instead of the default 7
.\backup.ps1 -Keep 14

# Restore from the most recent backup
.\backup.ps1 -Restore
```

**Schedule automated backups using Windows Task Scheduler:**

1. Open Task Scheduler → Create Basic Task
2. Name: `Akshaya Vistara Daily Backup`
3. Trigger: Daily, 2:00 AM
4. Action: Start a program
   - Program: `powershell.exe`
   - Arguments: `-NonInteractive -File "C:\path\to\akshaya_vistara\backup.ps1"`
5. Finish

## 11. PWA — Install as Desktop App

Akshaya Vistara can be installed as a native-looking desktop app on Windows 11
(no separate download needed — uses the browser's built-in PWA support).

### Generate icons first

```powershell
python manage.py generate_pwa_icons
python manage.py collectstatic
```

### Install in Edge (recommended on Windows 11)

1. Open http://localhost (or http://127.0.0.1:8000) in **Microsoft Edge**
2. Click the `...` menu (top-right) → **Apps** → **Install this site as an app**
3. Click **Install**

Akshaya Vistara now appears in your Start Menu and taskbar like a native app.

### Install in Chrome

1. Open the app URL in **Google Chrome**
2. Click the install icon (⊕) in the address bar, or go to `...` → **Cast, save, and share** → **Install page as app**

### What the PWA provides

- Standalone window (no browser UI)
- Works offline (recently visited pages are cached)
- App shortcuts: New Voucher, Scan Bill, P&L Report
- Push notifications (future feature)
- Windows Start Menu integration

## 13. Feature Reference

### Multi-Tenancy
- Every model has a `company` FK — queries are always filtered per company
- Users can access multiple companies with different roles (Admin/Accountant/Viewer)
- Active company stored in session; middleware enforces selection
- Switch companies via the navbar dropdown at any time

### Double-Entry Bookkeeping
- Every Voucher requires at least two VoucherItems
- ΣDebit must exactly equal ΣCredit — enforced in the view AND at model level
- Imbalanced vouchers are rejected with a clear error message
- Live balance counter in the Voucher form (JavaScript)

### Voucher Number Format
- Pattern: `{SHORTCODE}{FY}-{SEQUENCE:05d}`
- Example: `ABC2425-00001` (Company short code + FY 2024-25 + sequence)
- Thread-safe: uses `select_for_update()` to prevent duplicate numbers
- Resets per company per financial year

### OCR Bill Automation
1. Upload JPG/PNG/PDF invoice
2. Tesseract extracts raw text
3. Regex extracts: Vendor Name, GSTIN, Date, Total Amount
4. Auto-matches to existing ledger accounts
5. Detects duplicate bills (same GSTIN + amount)
6. Side-by-side preview + editable form
7. One-click creates balanced Purchase Voucher

### Reports
| Report | Description |
|--------|-------------|
| Profit & Loss | Income vs Expense for any date range; Net Profit/Loss |
| Balance Sheet | Assets vs Liabilities + Retained Earnings as of any date |
| Receivables Aging | Outstanding Sales by age bucket (0–30, 31–60, 61–90, 90+) |

### Dashboard
- Stat cards: Total Vouchers, Ledgers, Assets, OCR Pending
- Cash Flow line chart (last 12 months, Inflow vs Outflow)
- Balance by group donut chart
- Quick action buttons
- Recent vouchers with overdue due date highlighting

## 14. Troubleshooting

### "No module named 'django'" or "command not found"

The virtual environment is not active. Run:
```powershell
.\.venv\Scripts\Activate.ps1
```

### "SECRET_KEY not found" or "django.core.exceptions.ImproperlyConfigured"

The `.env` file is missing or incomplete. Run:
```powershell
Copy-Item ".env.example" ".env"
# Then edit .env and set a proper SECRET_KEY
```

### OCR returns empty text / wrong characters

1. Verify Tesseract is installed: `tesseract --version`
2. Check `TESSERACT_PATH` in `.env` matches actual installation path
3. Improve scan quality: good lighting, straight angle, 300 DPI minimum
4. For PDFs: verify Poppler is installed: `pdfinfo --version`

### "CSRF verification failed" after deployment

Add your domain to `ALLOWED_HOSTS` in `.env`:
```
ALLOWED_HOSTS=yourdomain.com,www.yourdomain.com
```

### Docker: "port 80 is already in use"

Another process is using port 80. Either stop it or change Nginx port in `docker-compose.yml`:
```yaml
ports:
  - "8080:80"   # Use 8080 on host instead
```

### Docker: database migrations not running

```powershell
docker-compose exec web python manage.py migrate
```

### Voucher "not balanced" error

Check that the sum of all **Debit** amounts equals the sum of all **Credit** amounts.
The live balance counter (green = balanced, red = unbalanced) in the form shows the difference.

### Balance Sheet shows a difference (not balanced)

This means some voucher items are on ledgers without matching double-entries.
Check for:
- Vouchers manually created in admin without VoucherItems
- Ledgers with opening balances that aren't offset
